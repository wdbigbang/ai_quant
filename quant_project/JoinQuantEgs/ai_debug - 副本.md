## 四、关键技术要点

⚠️ **重要提示**：以下技术要点在开发过程中必须严格遵守，违反可能导致策略运行失败或数据错误。

### 4.1 实盘兼容性要求

#### 1. 避免未来数据
**规则**：所有开盘前（09:25-09:30）的数据获取，必须使用昨日数据

```python
# ✅ 正确
price_data = get_price(
    security_list,
    end_date=context.previous_date,  # 使用昨日数据
    count=1,
    frequency='daily',
    fields=['open', 'close', 'high', 'low'],
    panel=False
)

# ❌ 错误
price_data = get_price(
    security_list,
    end_date=context.current_dt,  # 错误：盘前时这是当天
    count=1,
    frequency='daily',
    fields=['open', 'close', 'high', 'low'],
    panel=False
)
```

#### 2. context.current_dt 使用
```python
if hasattr(context.current_dt, 'hour'):
    hour = context.current_dt.hour
    minute = context.current_dt.minute
else:
    ts = pd.Timestamp(context.current_dt)
    hour = ts.hour
    minute = ts.minute
```

#### 3. Panel对象问题
```python
# ✅ 正确：返回 DataFrame 对象
price_data = get_price(security, panel=False)

# ❌ 错误：会返回已弃用的 Panel 对象，产生警告
price_data = get_price(security)
```

### 4.2 聚宽平台API注意事项

#### 4. get_current_data() 缓存机制（⚠️ 非常重要）

**缓存规则**：
- `get_current_data()` 在**同一定时任务**中有缓存机制
- **缓存作用域**：基于 `run_daily`、`run_weekly`、`run_monthly` 定时任务级别
- 无参数调用会缓存返回的股票列表，后续调用无法覆盖

**⚠️ 关键发现**：缓存无法在同一定时任务内覆盖！

**解决方案**：统一获取所有需要的数据，然后传递给子函数

### 4.3 交易日处理

#### 5. 交易日与日历日的区别
**规则**：时间相关判断应基于"第N个交易日"而非"第N天"

**实现位置**：`_check_capital_pool_adjustment()` 函数（已修复）

**状态**：✅ 已修复，从"每月1号"改为"每月第一个交易日"

### 4.4 交易记录处理

#### 6. 避免None值导致ERROR
**问题**：交易失败后trade_record为None，但被添加到g.today_trades，导致后续处理时ERROR

**解决方案**：只在交易成功时添加到g.today_trades

**状态**：✅ 已修复

### 4.5 持仓成本信息深拷贝（⚠️ 重要）

#### 7. 持仓引用问题（调试踩坑）
**问题**：卖出时无法计算盈亏，出现"无持仓信息，无法计算盈亏"WARNING

**根本原因**：下单前保存的持仓信息只是引用，下单后持仓被清空，引用的对象也被修改。

**解决方案**：深拷贝持仓成本信息的关键属性（avg_cost, total_amount, cost_value）。

**状态**：✅ 已修复

### 4.6 持仓标签管理（⚠️ 重要）

#### 8. 策略标签系统（调试踩坑）
**问题**：ETF被错误地标记为其他策略的持仓，导致持仓混乱和策略冲突。

**解决方案**：
1. 所有买入操作后必须设置策略标签：`g.strategy_tags[code] = self.strategy_id`
2. 所有卖出操作后必须删除策略标签：`del g.strategy_tags[code]`
3. 卖出前必须检查策略标签，只卖出属于当前策略的持仓

**状态**：✅ 已修复

### 4.7 小市值波段策略特有技术点

#### 9. 流通市值动态计算
**规则**：使用昨日财务数据+当前价格计算当前流通市值近似值

#### 10. 涨停股票过滤
**规则**：剔除当前涨停的股票，避免追高

#### 11. 冷静期天数计算
**规则**：按交易日计算冷静期天数，不是日历日

### 4.8 ETF轮动策略特有技术点

#### 12. 动量因子计算
**规则**：使用线性加权回归计算年化收益率和判定系数的乘积

#### 13. 空仓处理
**规则**：如果所有ETF得分都不在安全区间，则空仓

## 五、已解决的技术问题（调试踩坑记录）

### 5.1 v8.1.0 关键问题修复（2026-03-09）

#### 1. BaseStrategy交易委托失败问题（⚠️ 重要）
**问题描述**：6月4日出现"下单委托失败！下单数量为 56 (最小交易数量 100)"错误。

**根本原因**：买入前没有检查100股最小交易单位，导致下单数量不足100股被拒绝。

**解决方案**：在`buy_stocks()`方法中添加100股检查逻辑，计算预期股数，如果不足100股则调整买入金额或跳过。

**修改位置**：第5118-5131行

**状态**：✅ 已修复

#### 2. BaseStrategy订单管理问题（⚠️ 重要）
**问题描述**：失败订单没有被记录到交易统计中，无法追溯失败原因。

**根本原因**：`safe_order()`函数只在订单成功时添加交易记录，失败订单被忽略。

**解决方案**：在`safe_order()`函数中，当下单失败时也创建失败交易记录添加到`g.today_trades`，并标记为`status: 'failed'`。

**修改位置**：第2826-2855行

**状态**：✅ 已修复

#### 3. SwingStrategy冷静期计算错误（⚠️ 重要）
**问题描述**：日志中出现大量`[冷却期检查] 计算失败 xxx: module 'datetime' has no attribute 'strptime'`错误。

**根本原因**：文件开头导入`import datetime as dt`，但代码中使用`datetime.strptime()`，导致Python找不到该属性。

**解决方案**：将两处`datetime.strptime()`改为`dt.datetime.strptime()`（第4701行和第5694行）。

**修改位置**：第4701行、第5694行

**状态**：✅ 已修复

#### 4. SwingStrategy持仓混乱问题（⚠️ 重要）
**问题描述**：ETF(159915.XSHE)被错误地标记为test_strategy_1的持仓，导致持仓混乱。

**根本原因**：
1. SwingStrategy冷静期清仓没有检查策略标签，误删了属于ETF轮动策略的ETF
2. 货币基金买入时没有设置策略标签

**解决方案**：
1. 在`check_and_clean_stocks_in_cooldown()`方法中添加策略标签检查，只卖出属于当前策略的持仓
2. 在买入货币基金后设置正确的策略标签

**修改位置**：第4762-4793行、第4795-4803行

**状态**：✅ 已修复

#### 5. SwingStrategy失效问题（⚠️ 重要）
**问题描述**：6月24日后SwingStrategy完全失效，不买卖股票。
